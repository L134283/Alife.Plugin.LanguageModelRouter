# 灵枢 - 实时切换语言模型/报错自动切换容灾

## 核心功能

- **💬 一句话让 AI 切换渠道**：在聊天中告诉桌宠「切换到第二组」「切到 deepseek」，AI 自动调用切换函数，无需手动操作
- **🔄 手动一键切换**：UI 面板直接点击按钮切换到任意一组渠道，即刻生效
- **🛡️ 自动容灾切换**：遭遇 HTTP 429/402/5xx 或响应体含自定义关键字时，自动切换备用渠道重试
- **🔍 探测可用模型**：点击「探测」按钮拉取 `/v1/models` 获取该组渠道的完整模型列表，下拉选择后自动填入 Model ID
- **⚙️ 自动容灾开关**：可在容灾设置中开启/关闭自动切换，关闭后只使用当前选定组
- **📋 各组独立配置**：每组可单独设置 Reasoning Effort、Extra Headers、Extra Body

## 使用方式

1. 在角色配置中**禁用**「OpenAI语言模型」
2. **启用**「灵枢 - 实时切换语言模型/报错自动切换容灾」
3. 填写第 1 组（主渠道）必填项：Endpoint、Model ID、API Key
4. 可选填写第 2~4 组备用渠道，设置组名称（如 deepseek、京东）供 AI 识别
5. 配置保存后**即刻生效**，无需重载

### 聊天中切换

对桌宠说：
- 「切换到第二组」
- 「切到 deepseek」（按组名称匹配）
- 「换到京东」

AI 会调用 SwitchModelGroup 函数直接切换，无需手动操作 UI。

### 手动切换

面板底部「手动切换」区域点击对应按钮即可。

### 探测模型

每组 Model ID 下方有绿色「ⓘ 探测第N组」按钮。点击后拉取该渠道 `/v1/models` 接口，成功后显示模型下拉列表，选择即自动填入 Model ID。

## 配置说明

| 字段 | 说明 |
|------|------|
| 组名称 | 给渠道取别名（如 deepseek、京东），AI 可通过名称切换 |
| Endpoint | API 端点 URL，如 `https://api.openai.com/v1` |
| Model ID | 模型标识，如 `gpt-4o`、`deepseek-chat` |
| API Key | 认证密钥 |
| Reasoning Effort | 推理强度，可选 `low`/`medium`/`high`，留空不设置 |
| Extra Headers | 额外请求头，JSON 格式 |
| Extra Body | 额外请求体，JSON 格式 |
| 错误关键字 | 响应体含此关键字时触发切换，逗号分隔 |
| 启用自动容灾 | 开启后当前组失败自动切换备用渠道 |
| 重试间隔 | 切换前等待时间（毫秒），默认 1000ms |
